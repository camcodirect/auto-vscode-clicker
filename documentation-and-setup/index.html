A lightweight Windows utility that automatically clicks buttons and confirms prompts in VS Code and Claude CLI — so you never have to babysit your workflow again.

Quick Start (5 Minutes)

Get up and running as fast as possible. Follow these steps in order.

Download & Extract

If you received a VSCodeAutoClicker.zip file, extract it to any folder (e.g. your Desktop or Documents). You should see a folder containing VSCodeAutoClicker.exe, config.json, and a templates folder.

If running from source instead: Make sure Python 3.10+ is installed, open a terminal in the project folder, and run:

pip install -r requirements.txt

Create Your First Template

You need a screenshot of the button you want auto-clicked.

  1. Open VS Code and find the button on screen
  2. Press Win + Shift + S to open the snipping tool
  3. Draw a tight rectangle around just the button
  4. Save the image as button.png inside the templates folder
Use the Windows Snipping Tool to capture just the button. Save as templates/button.png.

Launch the App

From .exe: Double-click VSCodeAutoClicker.exe

From source: Double-click clicker.pyw, or run in terminal:

python clicker.py

A small green/yellow circle icon will appear in your Windows system tray (bottom-right corner near the clock).

Look for the colored circle in your system tray. Yellow = paused, Green = active.

Start Monitoring

The app starts paused by default. To begin:

  • Press Ctrl + Alt + P on your keyboard, OR
  • Right-click the tray icon and click "Resume"

The icon turns green. The app is now scanning your screen and will click the button whenever it appears.

Pause or Quit at Any Time

These keyboard shortcuts work globally — even when the app is controlling your mouse:

ActionShortcut
Pause / ResumeCtrl + Alt + P
Stop & QuitCtrl + Alt + Q

That's it! You're up and running. Read on for detailed explanations of every feature.

First Run Walkthrough

Here's exactly what happens the first time you launch the app, and what each visual cue means.

App starts in the system tray (paused)

When you launch the app, it immediately goes to the system tray in the bottom-right corner of your screen. It does not open a window — it lives entirely in the tray.

The icon starts as a yellow circle, meaning the app is paused and waiting for you to start it.

Check if your template is loaded

Right-click the tray icon. You'll see a menu. Look at the status lines at the top:

  • "Click Template: OK" — Your templates/button.png was found. You're ready.
  • "Click Template: NOT SET" — No template image found. You need to create one (see Templates).

Resume to start scanning

Click "Resume" in the tray menu, or press Ctrl + Alt + P. The icon turns green and the app begins scanning your screen.

Watch it work

When the app finds a match on screen, it will:

  1. Move your mouse to the button
  2. Briefly hover (80ms)
  3. Click
  4. Flash the tray icon blue briefly
  5. Wait for the cooldown period before scanning again

Pause when you need to use your mouse

Press Ctrl + Alt + P at any time to instantly pause. The icon goes yellow. Press again to resume.

Emergency stop: If the app takes over your mouse and you can't click anything, press Ctrl + Alt + Q to immediately quit the app. You can also move your mouse to the very top-left corner of your screen (pixel 0,0) to trigger PyAutoGUI's built-in safety stop.

Key Concepts

Template Matching

The app uses OpenCV (computer vision) to find your button screenshot on the live screen. It compares pixel patterns, not exact pixels, so small variations in rendering are tolerated.

Cooldown

After clicking, the app waits a configurable number of seconds before clicking again. This prevents rapid double-clicks and gives the UI time to respond.

Confidence Threshold

A score from 0 to 1 that determines how closely the screen must match your template. Default is 0.8 (80%). Lower = more lenient, higher = more strict.

Grayscale Mode

By default, images are converted to grayscale before matching. This is ~3x faster and works well for most buttons. Disable only if you need to match by color.

The 4 Modes

The app has four different modes for different automation scenarios. You can switch modes at any time through the tray menu or settings.

AUTO CLICK

Auto Click — Screen Button Clicker

Watches your screen for a specific button (matched by your template image) and clicks it automatically.

Best forClicking VS Code popups, dialog buttons, or any recurring on-screen button
RequiresA template screenshot of the button (templates/button.png)
How it actsMoves mouse to button center → hovers 80ms → clicks
Window visibilityThe button must be visible on screen
AUTO CONFIRM

Auto Confirm — Claude CLI Confirmer

Watches for Claude CLI confirmation prompts (the numbered selection dialog) and automatically selects "1. Yes".

Best forAutomatically approving Claude Code CLI prompts
RequiresOne or more confirm template screenshots in templates/confirm/
How it actsClicks the console area → types "1" → presses Enter
Window visibilityThe terminal window must be visible on screen
BOTH

Both — Click + Confirm Together

Runs Auto Click and Auto Confirm simultaneously. If both a button and a confirm prompt appear, the confirm prompt takes priority.

Best forComplex workflows that involve both UI buttons and CLI prompts
RequiresBoth click template and confirm templates
PriorityConfirm prompts are checked first, then click templates
BACKGROUND CONFIRM

Background Confirm — Works Minimized

Reads terminal text directly from the Windows console buffer using the Win32 API. Works even when the terminal is minimized or behind other windows.

Best forHands-free Claude CLI approval while working on other things
RequiresNothing — no template images needed
How it actsReads console text → detects prompt patterns → sends keystrokes directly
Window visibilityNot required! Works with minimized windows
Supported terminalscmd.exe, powershell.exe, pwsh.exe, bash.exe

Background Confirm detects these patterns in your terminal: > 1. Yes, [y/N], and (y/n). It also requires context phrases like "Do you want to proceed" or "requires approval" to prevent false positives.

Templates — Teaching the App What to Click

Templates are small screenshot images that tell the app what to look for on your screen. Think of them as "wanted posters" for buttons.

Creating a Click Template

Find the button on screen

Open VS Code (or whatever app has the button) and make the button visible.

Take a tight screenshot

Press Win + Shift + S and draw a rectangle around just the button. Don't include extra space around it.

Save the image

Method A (manual): Save it as button.png in the templates folder.

Method B (from tray): Right-click the tray icon → "Set Click Template..." → pick your screenshot file. It will be copied into the right place automatically.

Creating Confirm Templates

Confirm templates work the same way, but you can have multiple of them (to match different prompt styles).

Screenshot the confirm prompt

When Claude CLI shows a confirmation prompt, capture just the text that says > 1. Yes or similar.

Save to the confirm folder

Method A (manual): Save your screenshot(s) in the templates/confirm/ folder. Any name works.

Method B (from tray): Right-click the tray icon → "Add Confirm Template..." → pick your file.

You can add as many confirm templates as you want. The app tries each one until it finds a match.

Template tips: Capture at the same display scaling (100%, 125%, etc.) you normally use. If you change your display scaling, re-capture your templates. Templates captured in dark mode won't match a light mode UI.

Config File Reference

All settings are stored in config.json in the app folder. You can edit this file directly with any text editor (Notepad, VS Code, etc.).

{
    "mode": "auto_click",
    "template_path": "templates/button.png",
    "confirm_template_path": "templates/confirm_prompt.png",
    "confirm_templates_dir": "templates/confirm",
    "scan_interval_ms": 500,
    "confidence_threshold": 0.8,
    "click_cooldown_seconds": 3,
    "confirm_cooldown_seconds": 5,
    "monitor_index": 0,
    "region": null,
    "grayscale": true,
    "log_level": "INFO"
}

Fields not in the Settings dialog:

FieldDescription
template_pathRelative path to the click template image file
confirm_template_pathLegacy path for a single confirm template (still supported)
confirm_templates_dirFolder containing multiple confirm template images
regionSet to null for full screen, or {"left": 100, "top": 200, "width": 800, "height": 400} for a specific area

Be careful with JSON syntax! Missing commas or extra commas will cause the app to fail on startup. Use the Settings dialog if you're not comfortable editing JSON.

Background Confirm — Deep Dive

Background Confirm mode is the most advanced mode. It reads terminal text directly from Windows console memory buffers, without needing the terminal window to be visible.

How It Works

Find running terminal processes
cmd.exe, powershell.exe, pwsh.exe, bash.exe
Attach to each console buffer
via Win32 API (AttachConsole)
Read last 15 lines of text
ReadConsoleOutputCharacterW
Prompt pattern detected?
Send keystrokes to console
WriteConsoleInputW ("1" + Enter)

Detected Patterns

The app looks for these text patterns in the terminal output:

PatternResponse SentExample
> 1. Yes1 + EnterClaude CLI numbered selection
[y/N]y + EnterStandard yes/no prompt
(y/n)y + EnterAlternative yes/no format

Additionally, at least one of these context phrases must also appear to prevent false positives:

  • "Do you want to proceed"
  • "Do you want to allow"
  • "Do you want to make"
  • "requires approval"
  • "Esc to cancel"
  • "Yes, and don't ask again"

Limitation: Background Confirm works with traditional Windows console hosts (conhost.exe). Some modern terminals like Windows Terminal (wt.exe) use a pseudo-console (ConPTY) that may not be directly readable. If BG Confirm doesn't work, try running Claude CLI in a plain PowerShell or Command Prompt window.

Troubleshooting

The app isn't detecting my button

Most common fix: Lower the confidence threshold.

  1. Right-click tray → Settings
  2. Change Confidence Threshold from 0.80 to 0.65
  3. Click Save

If that's too sensitive (clicking wrong things), try 0.70 or 0.75.

Other causes:

  • Display scaling changed — If you changed your Windows display scaling (100% → 125%), recapture your template at the new scale.
  • Dark/Light mode changed — Templates from dark mode won't match light mode. Recapture.
  • Button is on wrong monitor — Check the Monitor Index setting.
  • Template includes too much background — Recapture with a tighter crop.

The app is clicking the wrong thing

  • Raise the confidence threshold to 0.85 or 0.90
  • Capture a more unique template — Include slightly more of the button's unique features
  • Use a scan region to limit where the app looks

High CPU usage

  • Increase the Scan Interval to 1000ms or higher
  • Enable Grayscale matching (it's ~3x faster)
  • Set a Scan Region instead of scanning the full screen
  • Set Monitor Index to a specific monitor instead of 0 (all)

The app won't start

  • Check that config.json is valid JSON (no missing commas, no trailing commas)
  • Check the clicker.log file for error messages
  • If running from source, make sure all dependencies are installed: pip install -r requirements.txt

Windows SmartScreen blocks the .exe

Click "More info" → "Run anyway". This is normal for unsigned applications and is safe.

Hotkeys don't work

  • Another application may be using the same hotkey combination
  • Try running the app as Administrator
  • Check clicker.log for "Failed to register hotkeys" messages

Background Confirm isn't detecting prompts

  • Make sure Claude CLI is running in cmd.exe, powershell.exe, pwsh.exe, or bash.exe
  • Windows Terminal (wt.exe) uses ConPTY which may not be supported. Use a plain PowerShell/Command Prompt window instead.
  • Set Log Level to DEBUG and check clicker.log for what text is being read from the console

Frequently Asked Questions

Q: Does this work on Mac or Linux?

No. This app is Windows-only because it uses Windows-specific APIs (Win32 Console API, system tray, DPI awareness). It requires Windows 10 or 11.

Q: Is this safe to use?

Yes. The app runs entirely locally on your computer. It never connects to the internet, collects data, or sends information anywhere. All it does is look at your screen and move your mouse.

Q: Will this trigger anti-cheat or security software?

The app uses standard mouse/keyboard automation (PyAutoGUI) and screen capture (mss). Most security software won't flag it, but some corporate endpoint protection tools might. If you encounter issues, check with your IT department.

Q: Can I run multiple instances?

Technically yes, but it's not recommended. Multiple instances would fight over mouse control. Use "Both" mode instead if you need multiple actions.

Q: How do I update the app?

Replace the .exe (or source files) with the new version. Your config.json and templates/ folder will be preserved as long as you don't delete them.

Q: Can I automate something other than VS Code?

Absolutely! The app works with any on-screen button in any application. Just capture a template of whatever button you want clicked. It's been used with VS Code, web browsers, dialog boxes, game UIs, and more.

Q: The mouse movement is interfering with my work. What do I do?

Press Ctrl+Alt+P to instantly pause the app whenever you need to use your mouse normally. Press it again to resume. Consider using Background Confirm mode instead, which doesn't move the mouse at all.
